prompt
  1. A message for the user.
 Opens a browser prompt with the specified message. Returns the response as a string if the user enters one.
  • robot.prompt('Enter your name:');
setId
  1. The number of the robot to be controlled.
  2. Optional: false to allow colliding into the ball.
Waits for roughly 1 second and then takes the number specified and sets that robot as the receiver of subsequent commands in the program. The number passed must be that of a robot present on the field. The robot will avoid running into the ball unless false is passed as a second argument.
  • robot.setId(0);
  • robot.setId(3, false);

                              		The robot will approach the ball from the current position set by the most recent aim command and then shoot. If no aim position was set, then center is assumed.

                              strikeLeft
                                		The robot will approach the ball from the current position set by the most recent aim command. If no aim position was set, then center is assumed. When it gets to the ball it will effectively aimLeft and shoot at the right corner. If the aim direction is not left, then there is a chance of missing the target; 30% if the aim direction is right and 10% if the direction is center.
                                • robot.strikeLeft();
                                strikeRight
                                  		The robot will approach the ball from the current position set by the most recent aim command. If no aim position was set, then center is assumed.                                                     Message Reason

                                                    Invalid robot number: <id> The argument is not a number in the range of 0 to 9.
                                                    Robot not found: <id> The robot with the specified number is not on the field.
                                                    Ball out of bounds. The ball is outside the field boundaries and cannot be interacted with.
                                                    Ball is moving too fast. When attempting to turn around the ball, the ball is moving too fast.
                                                    Robot must be facing the ball. When attempting to dribble or shoot the robot is not facing the ball.
                                                    <distance>mm is too far from ball; must be within 300. When attempting to dribble or shoot the robot is not close enough to the ball.
                                                    Ball is moving too slow. When attempting to catch the ball, the ball is moving too slow.
                                                    Robot is too close to ball. When attempting to catch the ball, the ball is too close to the robot.
